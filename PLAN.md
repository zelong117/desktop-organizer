### 1.3 关键发现

- **没有一个项目有AI能力** — 这是我们的差异化优势！
- 所有现有项目只做"分组显示"，不做"智能决策"
- C#/WPF是主流，但Python+Electron更适合AI集成
- 用户的真正痛点不是分组，而是"新文件该放哪"

## 二、用户痛点分析（基于真实桌面截图）

### 痛点清单（按严重度排序）

1. **🔴 项目文件散落桌面** — TAC28零件文件、DWG图纸、Excel报价单散落在桌面
2. **🔴 临时文件泛滥** — Hash命名的PNG、.bak、.err、.log无处归档
3. **🟡 重复文件** — 剑桥PPAP出现两次，外购回货单有宏/无宏两个版本
4. **🟡 编号体系维护难** — 00-99的18个大类，新文件不知道放哪个
5. **🟠 版本管理混乱** — V01/V02/(1)/(1)(1)版本叠加
6. **🟠 无智能建议** — 现有工具不帮用户决策

## 三、我们的产品：AI Desktop Organize（暂定名）

### 3.1 核心差异化（vs 所有竞品）

1. **AI智能分类** — 分析文件名+内容，自动建议放哪个文件夹
2. **项目感知** — 识别项目号（P019/TAC28/504000002145），自动归档
3. **去重检测** — 找出重复/相似文件，建议合并或删除
4. **临时文件清理** — 识别并归档临时文件（.bak/.err/.log/Hash名）
5. **一键整理预览** — 先看方案，确认后一键执行

**推荐方案A: Python+PyQt6**
- AI集成最方便（直接调OpenAI/本地模型）
- PyInstaller打包exe成熟
- 开发速度最快
- 可以直接用现有的hermes API调用
- Codex 5.5处理复杂逻辑没问题

## 四、开发计划

### Phase 1: 扫描引擎（1天）
- 扫描桌面所有文件
- 提取：文件名、大小、修改时间、类型
- 输出JSON格式的文件清单

### Phase 2: AI分类引擎（2天）  
- 基于文件名分析项目归属
- 基于文件类型建议目标文件夹
- 识别临时文件和重复文件
- 生成整理建议

### Phase 3: GUI界面（2天）
- 左右对比：当前状态 vs 整理后状态
- 可勾选/取消每个操作
- 拖拽调整分类

### Phase 4: 执行+撤销（1天）
- 一键移动文件
- 生成操作日志
- 支持撤销操作

### Phase 5: 打包发布（1天）
- PyInstaller打包exe
- 系统托盘图标
- 开机自启动选项

**总计：约7天，每天一个可运行版本**
